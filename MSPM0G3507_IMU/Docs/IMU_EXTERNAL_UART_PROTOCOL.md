# IMU 对外 UART 协议

外部 MCU UART：`UART2`，`PB15` 为 TX、`PB16` 为 RX，`460800 bps`、8N1。它独立于 Type-C 调试链路。

当前可编译选择两种输出：自定义格式默认 `100 Hz`，JY901 兼容格式默认 `10 Hz`。没有运行时修改发送频率的命令。

## 自定义格式

所有多字节字段为小端序。

```text
A5 5A LEN TYPE ID PAYLOAD... CRC_L CRC_H
```

- `LEN`：`TYPE + ID + PAYLOAD` 的字节数。
- `CRC`：对 `LEN` 至最后一个 `PAYLOAD` 字节计算 CRC16-CCITT；多项式 `0x1021`、初值 `0xFFFF`、无最终异或，低字节先发送。

### 命令帧

主机发送：`A5 5A 02 01 CMD CRC_L CRC_H`。命令均无 payload。

| CMD | 含义 |
|---:|---|
| `01` | Ping |
| `10` | 开始软件零飘标定 |
| `11` | 执行 XV7021BB AutoC |
| `12` | 开始 1080 度标定 |
| `13` | 当前积分角度归零 |

### 响应帧

设备返回：`A5 5A 06 02 CMD STATUS IMU_STATE CAL_RESULT 00 CRC_L CRC_H`。

`STATUS`：`00` 已接受，`01` CRC 错，`02` 帧错，`03` 未知命令，`04` 参数错，`05` 忙，`06` 失败。

该响应只表示命令是否被接受；校准完成与否应继续读取数据帧的状态。

### 数据帧

设备发送：`A5 5A 16 03 01 PAYLOAD[20] CRC_L CRC_H`，完整长度固定为 27 字节。

| Payload 偏移 | 类型 | 字段 | 单位/说明 |
|---:|---|---|---|
| 0 | `uint32` | `timeMs` | 系统毫秒计数 |
| 4 | `int32` | `angularRateMilliDps` | `0.001 deg/s` |
| 8 | `int32` | `angleMilliDeg` | `0.001 deg`，连续积分角 |
| 12 | `int32` | `normalizedAngleMilliDeg` | `0.001 deg`，范围 `[-180, 180)` |
| 16 | `uint8` | `imuState` | 见下表 |
| 17 | `uint8` | `appState` | 见下表 |
| 18 | `uint8` | `calResult` | 见下表 |
| 19 | `uint8` | `flags` | 见下表 |

| 字段 | 取值 |
|---|---|
| `imuState` | `0` 未初始化，`1` 就绪，`2` 零飘标定，`3` 1080 标定，`4` 错误 |
| `appState` | `0` 加热，`1` 温度稳定，`2` 零飘标定，`3` 正常发送，`4` 1080 标定 |
| `calResult` | `0` 无，`1` 进行中，`2` 成功，`3` 参数错，`4` 传感器错，`5` 静止不稳，`6` 超时，`7` 比例异常 |
| `flags` | bit0：静止自学习；bit1：RX 曾溢出；bit2：角度有效 |

本次上电完成零飘标定前，数据帧仍会发送，但角速度和角度字段为 0，且 `flags.bit2 = 0`。

## JY901 兼容格式

每帧固定 11 字节：

```text
55 TYPE D0L D0H D1L D1H D2L D2H D3L D3H SUM
```

`SUM` 为前 10 字节逐字节相加后的低 8 位。

| TYPE | 有效字段 | 换算 |
|---:|---|---|
| `52` | `D2` = Z 轴角速度 | `int16(D2) / 32768 * 2000 deg/s` |
| `53` | `D2` = 航向角 | `int16(D2) / 32768 * 180 deg`，范围 `[-180, 180)` |

其余字段目前填 0。设备每 100 ms 连续发送一帧 `52` 和一帧 `53`；本次上电校准成功前不发送 JY901 数据帧。JY901 输出模式下，设备仍接受上面的自定义命令帧，并以自定义响应帧答复。
