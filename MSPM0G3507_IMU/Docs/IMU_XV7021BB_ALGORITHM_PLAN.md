# XV7021BB IMU 已完成记录与后续优化

本项目当前使用 Epson XV7021BB 单轴陀螺仪。

参考手册：`E:\one_drive\OneDrive - CUHK-Shenzhen\diansai\Doc\IMU\XV7021BB_en.pdf`

## 器件参数

- 量程：`+/-400 deg/s`。
- 16-bit 角速度比例：`70 LSB/(deg/s)`。
- 24-bit 角速度比例：`17920 LSB/(deg/s)`。
- 12-bit 温度输出比例：`16 LSB/deg C`，25 deg C 典型输出约 `400 LSB`。
- 启动时序：串口通信至少等待 `1 ms`，温度数据至少等待 `80 ms`，角速度数据至少等待 `200 ms`。
- 典型零偏温漂：`0.0016 (deg/s)/deg C`。
- 典型角度随机游走：`0.065 deg/sqrt(h)`。
- 4-wire SPI 时钟上限：`10 MHz` when `VDDI > 2.4 V`，`5 MHz` when `VDDI <= 2.4 V`。
- 角速度输出 ODR：约 `13.770 kHz` 或 `14.160 kHz`，取决于芯片频率代码。
- 默认数字低通滤波为 `100 Hz`。角度积分场景下 HPF 保持关闭，避免低频角度信息被滤掉。

## 已完成内容

- 已将驱动命名从 `XV7011` 切换到 `XV7021`，后续指标按 XV7021BB 手册处理。
- `XV7021_Init()` 默认配置为 4-wire SPI、24-bit 角速度输出、12-bit 温度输出，并等待 200 ms 角速度启动时间。
- 新增 24-bit 原始角速度读取接口，角速度按 `17920 LSB/(deg/s)` 转换为 `deg/s`。
- 新增 `User/imu_task.c` 和 `User/imu_task.h`，作为 IMU 解算任务接口与实现。
- 当前 IMU 任务每 `500 us` 读取一次角速度，即 `2000 Hz` 目标采样；每 `50 ms` 读取一次温度；上位机打开上报后每 `10 ms` 发送一次 IMU 摘要帧。
- 角度解算采用软件零偏扣除、360 度比例修正、梯形积分，并使用 SysTick 推导出的微秒时间戳提升积分时间精度。
- 零飘标定使用 Welford 在线统计均值和方差，只有静止窗口满足绝对角速度和方差阈值时才接受结果。
- 360 度标定流程已实现：开始前静止采样更新零偏，旋转阶段积分角度，结束后静止确认，再计算 `scale_correction = 目标角度 / 实测角度`。
- 零飘标定已加入 `1000 ms` 进入状态后的静置等待，防止按键或上位机触发瞬间的机械震动进入零偏统计窗口；等待结束后才开始完整 `20000 ms` 零飘采样。
- 已接入 app 状态机：短按按键触发零飘标定，长按按键触发一圈 360 度标定。
- 已扩展 host link，上报 `IMU,S`、`IMU,CAL`、`IMU,ERR` 三类调试帧。
- 上位机已支持 `IMU_ZERO` 和 `IMU_CAL360` 命令，效果等同实体短按零飘和长按 360 度标定；HTML 页面已加入 RM 长方体方向模型，按 IMU 角度帧旋转显示方向。
- 已增加 `BSP_GetTickUs()`，用于高精度积分时间基准。
- 已新增 Flash 标定参数持久化：预留主 Flash 最后 `1 KiB`，地址 `0x0001FC00` 到 `0x0001FFFF`，只保存最新一次零飘和 360 标定参数；记录带 magic、版本、序号、温度、样本数和 CRC32。开机后校验通过则直接加载 `biasRaw24` 和 `scaleCorrection`。
- 已新增静止锁零试验版：根据 `mspm0_monitor_2026-07-15T10-20-17-529Z.csv` 的静止噪声统计，ready 状态下使用 `200 ms` 窗口判断静止；静止成立时仅把积分用角速度置零，避免静止噪声继续积成角度漂移。
- 已用 `rg` 确认没有 `XV7011/7011` 残留命名；已用 `cppcheck --check-level=exhaustive` 检查本次相关 C 文件。

## 当前实现默认值

- 角速度采样周期：`500 us`，目标 `2000 Hz`。
- 温度采样周期：`50 ms`。
- IMU 上报周期：`10 ms`，保持上位机链路负载不继续提高。
- 陀螺仪 SPI SCLK：BSP 运行时配置为 `5 MHz`，兼容 `VDDI <= 2.4 V` 的手册上限；若确认 `VDDI > 2.4 V` 且波形干净，可再评估 `10 MHz`。
- XV7021BB LPF：4 阶，`100 Hz`。
- XV7021BB HPF：关闭。
- 零飘触发后静置等待：`1000 ms`，等待结束后才开始统计零飘数据。
- 零飘默认采样窗口：`20000 ms`。
- 零飘静止判据：样本数不少于 `64`，最大绝对角速度不超过 `5 deg/s`，方差不超过 `0.25 (deg/s)^2`。
- 360 标定静止确认时间：`1000 ms`。
- 360 标定超时：`30000 ms`。
- 360 标定饱和保护：原始角速度绝对值达到 `380 deg/s` 视为接近饱和。
- 360 标定比例修正允许范围：`0.80` 到 `1.20`。
- 静止锁零窗口：`200 ms`；进入静止阈值为平均绝对角速度 `< 0.025 deg/s`、方差 `< 0.001225 (deg/s)^2`、最大绝对角速度 `< 0.10 deg/s`；退出静止阈值为单点绝对角速度 `> 0.12 deg/s` 或窗口平均绝对角速度 `> 0.06 deg/s`。

## 后续性能与精度优化

- SPI DMA 采样：当前 `BSP_SPI_GyroTransfer()` 已改为 DMA 搬运、同步等待完成，默认使用 DMA 通道 4/5，对应 SPI0 RX/TX 触发；SPI SCLK 已从 SysConfig 默认 `1 MHz` 在 BSP 初始化时提升到 `5 MHz`。下一步若要进一步降低调度抖动，应新增 `BSP_SPI_GyroTransferAsync()`，用 DMA 完成中断记录微秒时间戳，并把样本放入小型 ring buffer。
- 固定点与 MATHACL/IQMath：当前 float 实现便于验证。若确认 CPU 占用偏高，可把 raw LSB、bias、scale、angle 改成 Q 格式定点；方差、温度补偿、多项式拟合、批量 MAC、sqrt/atan 等计算优先评估 MSPM0 MATHACL 或 TI IQmath mathacl。
- 减少除法：高频路径里把 `1 / 17920`、`1e-6`、`scaleCorrection / 17920` 预计算成乘法系数，避免每次采样做浮点除法。
- 温度补偿：本项目已有温控策略，且目标温差控制较好，因此温度补偿优先围绕目标工作温度做窄温区优化。零飘参数应绑定温控稳定后的温度点保存；开机后先等待温控进入稳定窗口，再加载或确认零偏。只有温度偏离目标窗口明显时，才启用 `bias = b0 + kT * (T - T0)` 这类线性补偿或提示重新标定。
- 双向 360 标定：目前单次一圈即可计算比例。后续建议支持顺时针和逆时针各转一圈并取平均，降低人工起停、安装偏心和端点静止误差。
- 多圈标定：支持 `turns > 1` 的标定入口已经保留。实际标定时用 2 到 5 圈可以把起停误差摊薄，提高比例因子精度。
- 软件滤波：保留 XV7021BB 内部 100 Hz LPF 作为默认值。若实际运动频带更低，可增加轻量 IIR/FIR 或基于静止检测的自适应低通；注意不要滤掉真实低频角速度。
- 静止检测优化：现有静止判据基于最大角速度和方差。后续可加入温度稳定性、连续异常点剔除、median/MAD 等鲁棒统计，避免偶发抖动破坏标定。
- 时间戳验证：用逻辑分析仪或 GPIO 翻转测量 `IMU_Task_Run()` 实际采样间隔，确认 `500 us` 目标周期和抖动。如果调度抖动明显，应把采样触发迁移到定时器中断或 DMA 完成事件。
- 上报降载：当前 IMU 上报 `100 Hz`，不跟随陀螺仪内部采样继续上调。若后续 host 帧影响实时性，应支持 `SUB IMU <period_ms>` 或二进制帧，减少 ASCII 格式化和 UART DMA 竞争。
- 参数持久化后续优化：当前已经用最后 `1 KiB` 主 Flash 保存最新一次标定记录。后续若要提高断电写入鲁棒性，可升级为双槽/轮换写入，并记录写入状态字；如果标定频率很高，再评估 wear leveling。

## 温度补偿计划

1. 温控优先：IMU 解算应以温控稳定后的目标温度作为默认工作点。上电后如果温控还未进入稳定窗口，可以先输出未完全可信状态，避免把冷机零偏当作长期零偏使用。
2. 建立基准：把一次可信零飘标定时的温度记为 `T0`，零偏记为 `bias0_raw24`。由于温差控制较好，正常运行时优先直接复用这个零偏，而不是频繁动态改写。
3. 窄窗口判断：保存并检查 `T0` 附近的允许温差窗口，例如 `abs(T - T0) <= deltaT_valid`。窗口内直接使用持久化零偏；窗口外再启用补偿或提示重新标定。
4. 一阶补偿作为兜底：若实测发现温控目标附近仍有可见漂移，再用线性模型 `bias_raw24(T) = bias0_raw24 + k_raw24_per_c * (T - T0)`。手册典型值只作为初值，最终以实测拟合为准。
5. 静止自学习谨慎开启：当静止检测连续通过且温控稳定时，可以用当前 raw24 均值慢慢修正零偏；运动中不更新曲线，防止把真实转动误学成零漂。
6. 分段/二阶升级：只有当目标温区外也要求高精度，且一阶线性残差明显时，再升级为分段线性表或二阶多项式。拟合和表查可以后续迁移到 MATHACL/IQMath。
7. 持久化：保存 `T0`、`bias0_raw24`、`deltaT_valid`、`k_raw24_per_c`、拟合质量、温控目标温度和 CRC。开机后加载，若当前温度未进入有效窗口，则等待温控稳定或提示重新标定。
